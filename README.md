## 📖 О проекте

Веб-сайт для аренды одежды – дипломный проект по курсу «Python-разработчик».  
Пользователи могут размещать объявления об аренде одежды, искать их по различным фильтрам, оставлять отзывы, отправлять запросы на аренду и имитировать оплату.  
Объявления проходят модерацию перед публикацией.

Проект реализован на **Django 5.1** с использованием **PostgreSQL 15**, **Bootstrap 5** и **Docker**.

---

## 🚀 Основной функционал

- **Пользователи**: регистрация, аутентификация, личный кабинет.
- **Объявления**: создание, редактирование, удаление (автор или администратор), загрузка фото, выбор категорий, указание периода доступности.
- **Поиск и фильтрация**: по ключевым словам, городу, цене, категориям.
- **Модерация**: одобрение или удаление объявлений сотрудниками (`is_approved`).
- **Отзывы**: текст, оценка (1–5), опциональное фото. Один пользователь – один отзыв на объявление.
- **Аренда**: запросы на аренду с выбором дат и количества, изменение статуса (одобрен/отклонён/отменён), имитация оплаты.
- **Пагинация**: на странице списка объявлений (12 шт.) и на странице отзывов (5 шт.).
- **Админ-панель**: управление пользователями, объявлениями, категориями, отзывами, запросами на аренду.

## 📦 Установка и запуск

### Требования
- Docker и Docker Compose
- Либо Python 3.12 и PostgreSQL 15 (для локального запуска)

### Запуск через Docker (рекомендуемый способ)

1. **Клонировать репозиторий**
   ```bash
   git clone https://github.com/ваш-логин/Python_Site_FinalProject_Kondratev.git
   cd Python_Site_FinalProject_Kondratev

2. **Создать файл .env (пример в .env.example)**
    ```bash 
    cp .env.example .env
    # при необходимости отредактировать пароли, ключи

3. **Запустить контейнеры**
    ```bash 
    docker-compose up --build -d
   
4. **Применить миграции (автоматически выполняются при запуске, но можно вручную)**
   ```bash 
   docker-compose exec web python manage.py migrate

5. **Создать суперпользователя (для админки /admin)**
    ```bash
   docker-compose exec web python manage.py createsuperuser

6. **Открыть сайт**
    ```bash
   http://localhost:8000
   
### Локальный запуск (без Docker)

1. **Создать и активировать виртуальное окружение**
    ```bash
    python -m venv venv
    source venv/bin/activate  # Linux/Mac
    venv\Scripts\activate     # Windows

2. **Установить зависимости**
    ```bash
   pip install -r requirements.txt
   
3. **Настроить базу данных PostgreSQL**
    ```bash
   Создать базу, пользователя, пароль. Внести данные в .env или в settings.py.
   
4. **Применить миграции**
    ```bash
   python manage.py migrate

5. **Запустить сервер**
    ```bash
   python manage.py runserver
   
### 🧪 Тестирование
**Тесты написаны с использованием pytest. Для запуска:**
```bash
docker-compose exec web pytest -v
```
**Или локально (если окружение активировано):**
```bash
pytest -v
```
